# План работы над проектом «Акинатор»

Старт: 15.04  
Дедлайн: 06.05  
Роли: Team Lead (TL), Backend (BE), Frontend (FE), Tester (TE)

## Этап 1 (15.04 – 19.04) — Инициализация и заглушки
- TL: создать репозиторий, `.gitignore` (C), структуру папок, Makefile (цели all, clean, test).  
- BE: создать `tree.h`, `tree.c` с пустыми функциями, возвращающими заглушки.  
- FE: создать `io.h`, `io.c` с заглушками, реализовать безопасный ввод строк.  
- TE: подготовить начальные файлы деревьев (`data/default_tree.txt`, `data/superheroes.txt`) на выбранную тему.

## Этап 2 (19.04 – 22.04) — Загрузка/сохранение дерева
- BE: реализовать `load_tree_from_file`, `save_tree_to_file`, `create_object_node`, `create_question_node`, `free_tree`.  
- TE: написать юнит-тесты для сериализации (проверить, что после сохранения и загрузки дерево идентично).

## Этап 3 (23.04 – 26.04) — Игровой цикл без обучения
- FE: дописать `ask_question`, `confirm_object` (валидация д/н).  
- BE: реализовать `play_round` (обход дерева до листа, возврат результата).  
- TL: интегрировать `main.c` (загрузка дерева, цикл «Сыграем ещё?», вызов `play_round`).  
- TE: ручное тестирование угадывания (всегда верный ответ).

## Этап 4 (27.04 – 30.04) — Обучение (добавление нового объекта)
- BE: реализовать `add_new_object`.  
- FE: реализовать `ask_real_object`, `ask_new_question`, `ask_answer_for_new`.  
- TL: связать `play_round` так, чтобы при неверной догадке вызывалось обучение и перезапись файла.  
- TE: написать тесты для добавления узла (проверить структуру дерева после обучения).

## Этап 5 (01.05 – 03.05) — Обработка ошибок и памяти
- Все вместе: проверить обработку некорректного ввода, повреждённого файла, отсутствия файла (создание дерева по умолчанию).  
- TL+BE: прогон Valgrind, исправление утечек.

## Этап 6 (04.05 – 06.05) — Финальное тестирование и документация
- TE: выполнить сценарии ручного тестирования (из ТЗ), подготовить отчёт о тестировании.  
- TL: оформить README (сборка, запуск, примеры, формат файла), запустить Doxygen, провести код-ревью.  
- Все: фиксация коммитов, создание тега `v1.0`. Презентация.
